# Resume Parser and Manager

A full-stack web application for intelligent resume parsing, AI-powered career insights, and smart candidate recruitment. Built with **React** (Frontend), **FastAPI** (Backend), and **MongoDB** (Database), powered by **Groq AI** for resume analysis and intelligent candidate ranking.

**Live Demo:** [https://resume-parser-and-manager.vercel.app](https://resume-parser-and-manager.vercel.app)

---

## Features

### For Candidates

- **Resume Upload & Parsing** — Upload PDF, DOCX, or TXT resumes for automatic, comprehensive data extraction covering 20+ fields including contact info, education, experience, skills, projects, certifications, publications, and more.
- **AI-Powered Career Insights** — Get personalized career recommendations, strengths analysis, skill gap identification, and interview tips generated by a large language model.
- **Resume History** — View and manage all previously uploaded resumes with the ability to delete entries.
- **Profile Management** — Update personal information, email, and credentials. A new JWT token is issued automatically if the email is changed.

### For Recruiters

- **Bulk Resume Upload** — Upload up to 50 resumes at once with built-in rate limiting, exponential backoff, duplicate detection, and per-file retry logic.
- **AI-Powered Chatbot** — Natural language candidate search with intelligent multi-criteria ranking. Ask questions like *"Find top 5 Python developers"* or *"Who has 5+ years of experience in machine learning?"*
- **Candidate Database** — Browse, filter, and delete parsed candidate profiles stored under the recruiter's account.
- **Smart Duplicate Detection** — Automatically identifies duplicate resumes based on a hash of email, name, and phone number.
- **Email Outreach** — Generate AI-drafted, personalized outreach emails for any candidate directly from the chatbot interface, with a one-click mailto link.

### AI Capabilities

- **Hybrid Resume Parsing** — Combines regex-based extraction (for high-confidence fields like email, phone, DOB, gender) with LLM-based extraction for structured sections.
- **Intelligent Skill Detection** — Extracts both explicitly listed skills and skills derived from project and experience descriptions.
- **Multi-Criteria Candidate Ranking** — Weighted scoring across skill proficiency, experience quality, project relevance, education level, and company background, adapted dynamically to query intent.
- **Conversational Memory** — The chatbot maintains conversation history within a session for follow-up queries.
- **Indian City-to-State Mapping** — Location-aware filtering using a comprehensive city-state mapping utility.

## Quick Start (Local Development)

### 1. Clone the Repository

```bash
git clone https://github.com/bismaysarangi/resume-parser-and-manager.git
cd bismaysarangi-resume-parser-and-manager
```

### 2. Backend Setup

```bash
cd server
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env
# Edit .env and fill in your MongoDB credentials, SECRET_KEY, and GROQ_API_KEY

uvicorn main:app --reload --port 8000
```

### 3. Frontend Setup

```bash
cd ../client
npm install
npm run dev
```

Visit **http://localhost:5173**

## Intelligent Candidate Ranking

The AI chatbot ranks candidates using five weighted scoring dimensions:

### Scoring Dimensions

**Skill Proficiency (0–100)**
- Explicit skill listing: 30 pts
- Skills used in projects: up to 40 pts
- Skills found in experience descriptions: up to 30 pts

**Experience Quality (0–75)**
- Total years of experience: up to 40 pts
- Company diversity: up to 15 pts
- Senior role indicators: up to 20 pts

**Project Relevance (0–100)**
- Number of relevant projects using required skills
- Project complexity indicators from descriptions

**Education Score (0–100)**
- PhD: 100 · Master's: 70–75 · Bachelor's: 50–70

**Company Background (0–100)**
- Recognition for top-tier or well-known employers

### Dynamic Weights by Query Intent

```python
# Skill-focused queries (e.g., "Find Python developers")
weights = {
    'skills': 0.45, 'projects': 0.30,
    'experience': 0.15, 'education': 0.05, 'company': 0.05
}

# Company/experience-focused queries
weights = {
    'company': 0.50, 'experience': 0.25,
    'skills': 0.15, 'projects': 0.05, 'education': 0.05
}

final_score = sum(score * weight for score, weight in zip(scores, weights.values()))
```

## Rate Limiting & Retry Logic

### Bulk Upload
- Base delay: 2 seconds between API calls
- Dynamic delay scaling based on requests in the current window
- Maximum 8 requests per minute (aligned to Groq's limits)
- Exponential backoff on rate limit errors: 10s → 20s → 40s → 60s
- Up to 5 retry attempts per resume

### Single Resume Upload
- Up to 3 retries with exponential backoff
- 90-second request timeout

### Chatbot
- Optimized for low-latency interactive use
- Smaller context windows for faster response times

## Troubleshooting

**MongoDB connection error** — Verify credentials in `.env`, ensure your IP is whitelisted in MongoDB Atlas (use `0.0.0.0/0` for Render), and confirm the cluster URL format.

**Groq rate limiting** — The built-in retry and backoff logic handles most cases automatically. Check your usage quota at [console.groq.com](https://console.groq.com).

**CORS errors** — In production, ensure `CORS_ORIGINS` env var on Render is set to `https://resume-parser-and-manager.vercel.app`. For local dev, it defaults to `http://localhost:5173`.

**Token expired** — JWT tokens expire after 8 hours. Clear `localStorage` and log in again.

**Render cold start** — The free tier spins down after inactivity. Use [UptimeRobot](https://uptimerobot.com) to ping `https://resume-parser-and-manager.onrender.com/` every 5 minutes to keep it alive.

## Example API Calls

```bash
# Register a candidate
curl -X POST https://resume-parser-and-manager.onrender.com/api/v1/auth/signup \
  -H "Content-Type: application/json" \
  -d '{"username":"jane","email":"jane@example.com","password":"secret123","role":"candidate"}'

curl -X POST https://resume-parser-and-manager.onrender.com/api/candidate/parse-resume \
```
